# AI 작업 제어 규칙

## 1. 카파시(Karpathy)의 4대 행동 제어 규칙

### 1.1 코딩 전 생각하기 (Think Before Coding)

지시사항이 모호할 경우 AI가 임의로 기저에 깔린 의도를 추측하거나 가정하지 못하게 한다. 코드 작성전에 반드시 사용자에게 먼저 질문하도록 강제한다.

### 1.2 단순성 우선 (Simplicity First)

지시하지 않은 기능, 미래를 대비한 불필요한 유연성, 오버엔지니어링을 추가하지 않는다. 문제를 해결하는 가장 단순한 코드만 작성한다.

### 1.3 외과적 수술 같은 변경 (Surgical Changes)

코드를 수정할 때 주변의 관련 없는 코드, 주석, 포맷팅을 임의로 개선하지 않는다. 요청한 작업에 해당하는 코드만 정확히 수정한다.

### 1.4 목표 기반 실행 (Goal-Driven Execution)

단순히 코드를 수정하는 데서 멈추지 않는다. 가능한 경우 에러를 재현하는 테스트를 먼저 작성하고, 성공 기준을 명확히 한 뒤 통과할 때까지 검증한다.

## 2. 환각(Hallucination) 원천 차단 규칙

AI가 실제로 확인하지 않은 내용을 알고 있는 것처럼 말하지 못하게 하기 위한 규칙이다.

### 2.1 기억 대신 도구 먼저 사용 (Tool-first, not memory-first)

파일 내용이나 프로젝트 구조에 대해 과거 기억이나 추측에 의존하지 않는다. 반드시 검색이나 파일 읽기 도구를 사용해 현재의 실제 코드를 확인한 후 작업한다.

### 2.2 연쇄 추측 금지 (No Chain-Guessing)

첫 번째 주장이 추측에 불과하다면 거기서 멈추고 확인부터 한다. 추측 위에 추측을 쌓아 그럴듯한 오답을 만들지 않는다.

### 2.3 출처 명시 강제 (Cite the Source)

코드 상태나 API에 대해 설명할 때는 어떤 파일의 몇 번째 줄을 확인했는지 명시한다. 출처를 댈 수 없다면 모른다고 말한다.

## 3. 프로세스 단계화 및 정지(Stop) 규칙

AI가 지시를 받자마자 임의로 코드를 작성하지 않도록 작업 단계를 강제한다.

### 3.1 탐색 → 계획 → 실행 순서 강제

바로 코딩하지 않는다. 먼저 관련 파일을 분석하고, 그다음 작업 계획(Task list)을 세운 뒤 실행한다.

작업 순서:

1. Explore: 관련 문서, 코드, API 규약, DB schema를 확인한다.
2. Plan: 변경 범위와 성공 기준을 정리한다.
3. Execute: 승인된 범위 안에서만 구현한다.
4. Verify: 테스트, 빌드, 문서 검증을 수행한다.
